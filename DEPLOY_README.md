# 部署说明

## 服务器信息
- **IP地址**: 175.178.193.103
- **用户名**: administrator
- **构建目录**: `./dist`

## 部署方式

### 方式1: 使用rsync部署（推荐，如果服务器支持SSH）

```bash
./deploy_rsync.sh
```

如果默认端口22失败，可以尝试其他端口：
```bash
./deploy_rsync.sh 2222
./deploy_rsync.sh 8022
```

**注意**: 执行时会提示输入密码，请输入 administrator 账户的密码。

### 方式2: 使用自动部署脚本

```bash
./deploy.sh
```

该脚本会自动尝试SSH和SMB两种方式。

### 方式3: 手动部署（如果自动方式都失败）

#### 选项A: 使用远程桌面
1. 使用远程桌面连接到 `175.178.193.103`
2. 用户名: `administrator`
3. 输入密码
4. 将 `dist` 目录的所有内容复制到 `C:\inetpub\wwwroot\`

#### 选项B: 使用压缩包
已为您创建了 `deploy.tar.gz` 压缩包，您可以：
1. 通过远程桌面或其他方式将压缩包上传到服务器
2. 在服务器上解压到 `C:\inetpub\wwwroot\`

#### 选项C: 使用FTP客户端
如果服务器启用了FTP服务，可以使用FileZilla等FTP客户端上传 `dist` 目录的内容。

## 文件位置

- **构建文件**: `./dist/`
- **压缩包**: `./deploy.tar.gz`

## 注意事项

如果自动部署失败，请检查：
1. 服务器是否启用了SSH服务（端口22或其他）
2. 服务器是否启用了SMB/CIFS共享服务
3. 防火墙是否允许相应端口的连接
4. 用户名和密码是否正确
